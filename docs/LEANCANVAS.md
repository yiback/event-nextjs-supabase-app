## 1. 문제 (Problem)

### 주최자 관점
- 카카오톡/문자로 참석 여부를 일일이 확인하는 것이 **번거롭고 시간 소모적**
- 참석 응답이 여러 채널에 분산되어 **취합이 어려움**
- 모임 정보와 공지가 대화 속에 묻혀 **확인하기 힘듦**
- 정기 모임의 반복적인 이벤트 생성과 알림 발송이 **비효율적**

### 참여자 관점
- 모임 일정과 장소를 매번 확인하기 **불편함**
- 참석 여부를 응답했는지 **기억하기 어려움**
- 이벤트 날짜가 다가오면 **리마인더가 없어서 놓치는 경우** 발생
- 여러 모임에 속해 있을 때 **일정 관리가 복잡함**

### 기존 대안
- 카카오톡 그룹 채팅 (정보 파편화)
- 네이버 밴드 (기능 과다, 복잡한 UI)
- 엑셀/스프레드시트 (비효율적, 실시간 업데이트 어려움)

## 2. 고객 세그먼트 (Customer Segments)

### 타겟 사용자
- **소규모 정기 모임 주최자** (5-10명 규모)
- **모임 참여자**

### 주요 모임 유형
- 🏊 수영, 헬스 등 **운동 동호회**
- 👫 친구 정기 모임
- 📚 독서, 스터디 모임
- 🎮 취미 동호회
- 🍽️ 맛집 탐방 모임

### 사용자 특성
- 주 1회 이상 정기적으로 모임을 진행
- 참석 인원 파악이 중요한 활동 (장소 예약, 비용 분담 등)
- 스마트폰 사용에 익숙한 20-40대
- 빠르고 간편한 서비스를 선호

### 얼리 어답터
- 모임을 직접 운영하는 **주최자/총무**
- 참석 관리에 어려움을 겪고 있는 사람

---

## 3. 가치 제안 (Unique Value Proposition)

### 핵심 가치

> **"소규모 모임의 참석 관리를 한 곳에서 간편하게"**

### 고유한 가치 제안

- 🎯 **원클릭 참석 응답**: 참석/불참/미정을 버튼 하나로 빠르게 응답
- 📊 **실시간 참석 현황**: 누가 참석하는지 한눈에 파악
- 🔔 **자동 리마인더**: 이벤트 전날 자동 알림으로 불참 방지
- 🔗 **간편한 초대**: 초대 코드 공유로 쉬운 모임 가입
- 📱 **모바일 최적화**: 언제 어디서나 간편하게 확인

### 상위 개념
> 소규모 모임의 **운영 효율성을 극대화**하는 참석 관리 플랫폼

## 4. 해결책 (Solution)

### 핵심 기능

| 기능 | 해결하는 문제 |
|------|--------------|
| **모임 생성 및 관리** | 모임 정보의 중앙 집중화 |
| **이벤트 CRUD** | 반복적인 일정 관리 자동화 |
| **참석 응답 시스템** | 참석 여부 취합의 번거로움 해소 |
| **실시간 참석 현황** | 누가 오는지 즉시 파악 |
| **웹 푸시 알림** | 이벤트 리마인더 자동 발송 |
| **초대 코드 시스템** | 간편한 모임 가입 프로세스 |
| **공지사항 기능** | 중요 정보 전달의 효율화 |

### 기술적 구현
- Next.js 15 + Supabase 기반 웹 애플리케이션
- PWA 지원으로 앱처럼 사용 가능
- 소셜 로그인 (카카오/구글)으로 가입 장벽 최소화

## 5. 경쟁우위 (Unfair Advantage)

### 차별화 요소

- **최적화된 사용자 경험**: 소규모 모임에 딱 맞는 심플한 기능
- **빠른 온보딩**: 소셜 로그인 + 초대 코드로 30초 내 가입 완료
- **실시간 동기화**: Supabase Realtime으로 즉각적인 상태 업데이트
- **PWA 지원**: 앱 설치 없이 웹으로 푸시 알림 수신

### 경쟁 서비스 대비 강점

| 비교 항목 | 카카오톡 | 네이버 밴드 | 본 서비스 |
|----------|---------|------------|----------|
| 참석 응답 | 수동 취합 | 일정 기능 | **원클릭 응답** |
| 실시간 현황 | ❌ | △ | **✅ 즉시 반영** |
| 리마인더 | ❌ | △ | **✅ 자동 발송** |
| 사용 복잡도 | 낮음 | 높음 | **최적화** |
| 정보 파편화 | 심함 | 보통 | **중앙 집중** |

## 8. 주요 지표 (Key Metrics)

### 핵심 성과 지표 (KPI)

| 지표 | 설명 | 목표 |
|------|------|------|
| **MAU** (월간 활성 사용자) | 월 1회 이상 접속 사용자 | 초기 100명 |
| **이벤트 생성 수** | 월간 생성된 이벤트 수 | 50개/월 |
| **참석 응답률** | 이벤트별 응답 완료 비율 | 80% 이상 |
| **푸시 알림 클릭률** | 알림 클릭 비율 | 30% 이상 |
| **리텐션** (주간) | 1주 후 재방문 비율 | 60% 이상 |

### 추적 지표

- **신규 모임 생성 수**: 서비스 성장 지표
- **모임당 평균 멤버 수**: 타겟 적합성 검증
- **평균 응답 시간**: 사용자 참여도
- **NPS (순추천지수)**: 사용자 만족도

## 9. 채널 (Channels)

### 사용자 획득 채널

| 채널 | 전략 |
|------|------|
| **입소문/바이럴** | 초대 코드 공유로 자연스러운 확산 |
| **SNS 마케팅** | 인스타그램, 페이스북 타겟 광고 |
| **커뮤니티** | 운동 동호회, 취미 모임 카페/밴드 홍보 |
| **앱스토어 최적화** | PWA 설치 가이드 및 검색 노출 |
| **콘텐츠 마케팅** | 모임 운영 팁 블로그/유튜브 |

### 핵심 성장 전략
> **초대 코드 기반 바이럴 루프**
>
> 주최자 → 모임 생성 → 초대 코드 공유 → 참여자 가입 → 새로운 모임 생성
